ToolShare Azure Ubuntu VM Deployment Checklist

Fallback deployment plan for this project when Azure Database for MySQL Flexible Server is blocked by the Azure for Students subscription. This path runs the PHP app and MySQL on one Ubuntu VM.

Recommended Path Ubuntu VM + Apache + PHP + MySQL
Best For Capstone demo with lowest platform risk
Project Fit Matches current local XAMPP-style PHP/MySQL app

Cost Estimate

These numbers are an estimate, not a quote. Azure’s pricing pages say VM and disk charges vary by region, agreement, and current pricing program. The ranges below are an inference for a small Linux VM plus a small managed disk, based on Azure’s official pricing pages and a demo-sized deployment.

Recommended Demo VM
Size: `Standard_B2s`
Disk: Standard SSD 32 to 64 GiB
Estimated monthly: about $35 to $50
Estimated 2-day demo: about $2.50 to $4.50
Cheapest Possible VM
Size: `Standard_B1s` if offered and stable enough
Disk: Standard SSD 32 GiB
Estimated monthly: about $15 to $25
Estimated 2-day demo: about $1 to $2
Important: VM compute is the main cost. Storage, light outbound traffic, and a tiny public IP cost are small compared with the VM itself. Since you have Azure student credit, this approach should still stay well inside demo budget unless you leave the VM running for a long time.

What You Need Before Starting

  1. Azure student subscription with enough credit.
  2. Your local project folder ready on your Mac.
  3. Your local MySQL data ready to export with `mysqldump`.
  4. Gmail SMTP app password ready for production email sending.
  5. Stripe test keys and webhook secret ready.

Step 1: Create the Ubuntu VM

  1. Open Azure Portal.
  2. Go to Virtual machines → Create → Azure virtual machine.
  3. Set:
    • Subscription: `Azure for Students`
    • Resource group: create `toolshare-vm-rg`
    • Virtual machine name: `toolshare-vm`
    • Region: choose one Azure allows for your subscription
    • Image: `Ubuntu Server 22.04 LTS`
    • Size: `Standard_B2s` recommended
    • Authentication type: `Password`
    • Username: `azureuser`
  4. Open inbound ports:
    • `SSH (22)`
    • `HTTP (80)`
    • `HTTPS (443)`
  5. Review and create the VM.

Step 2: Add a DNS Name

  1. Open the VM after creation.
  2. Click the attached Public IP address resource.
  3. Go to Configuration.
  4. Set a DNS label, for example `toolsharevidhi`.
  5. Save it. Your hostname will look like `toolsharevidhi.centralus.cloudapp.azure.com`.

Step 3: SSH Into the VM

From your Mac terminal, run:

ssh azureuser@toolsharevidhi.centralus.cloudapp.azure.com

Step 4: Install the Runtime

Run these commands on the VM:

sudo apt update sudo apt -y install apache2 mysql-server unzip composer php php-cli libapache2-mod-php php-mysql php-curl php-mbstring php-xml php-zip php-gd php-intl sudo systemctl enable --now apache2 sudo systemctl enable --now mysql
Project runtime match: this project already uses Stripe, Dompdf, PHPMailer, PDO MySQL, GD, XML, ZIP, cURL, and mbstring. The package list above covers that stack.

Step 5: Create the Database and DB User

sudo mysql -e "CREATE DATABASE tool_rental CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;" sudo mysql -e "CREATE USER 'toolshareapp'@'localhost' IDENTIFIED BY 'YourStrongDbPasswordHere';" sudo mysql -e "GRANT ALL PRIVILEGES ON tool_rental.* TO 'toolshareapp'@'localhost';" sudo mysql -e "FLUSH PRIVILEGES;"

Step 6: Export the Local Database

On your Mac, from the project environment, export your current local DB:

mysqldump -u root -p tool_rental > tool_rental.sql

Step 7: Create the Deployment ZIP

On your Mac, from the project root:

zip -r toolshare-vm.zip . -x "tool_rental.sql" "tmp/*" "uploads/*" "docs/*" ".env" "*.log" ".DS_Store"
Do not include: SQL dumps, temp files, docs, logs, or local-only secret files in the deployment ZIP.

Step 8: Upload the App and SQL Dump

From your Mac:

scp toolshare-vm.zip tool_rental.sql azureuser@toolsharevidhi.centralus.cloudapp.azure.com:~

Step 9: Unpack the App on the VM

sudo mkdir -p /var/www/toolshare sudo unzip -o ~/toolshare-vm.zip -d /var/www/toolshare sudo mkdir -p /var/www/toolshare/uploads/disputes sudo mkdir -p /var/www/toolshare/tmp/dompdf sudo chown -R azureuser:www-data /var/www/toolshare sudo chown -R www-data:www-data /var/www/toolshare/uploads /var/www/toolshare/tmp sudo chmod -R 775 /var/www/toolshare/uploads /var/www/toolshare/tmp

Step 10: Import the SQL Dump

mysql -u toolshareapp -p tool_rental < ~/tool_rental.sql

Step 11: Configure Apache

Create a site config:

sudo nano /etc/apache2/sites-available/toolshare.conf

Paste this template and replace the placeholders:

<VirtualHost *:80> ServerName toolsharevidhi.centralus.cloudapp.azure.com DocumentRoot /var/www/toolshare <Directory /var/www/toolshare> AllowOverride All Require all granted DirectoryIndex index.php index.html </Directory> SetEnv DB_HOST 127.0.0.1 SetEnv DB_PORT 3306 SetEnv DB_NAME tool_rental SetEnv DB_USER toolshareapp SetEnv DB_PASS YourStrongDbPasswordHere SetEnv DB_CHARSET utf8mb4 SetEnv MAIL_ENABLED true SetEnv MAIL_HOST smtp.gmail.com SetEnv MAIL_PORT 587 SetEnv MAIL_SECURE tls SetEnv MAIL_USERNAME yourgmail@gmail.com SetEnv MAIL_PASSWORD your_16_character_google_app_password SetEnv MAIL_FROM_EMAIL yourgmail@gmail.com SetEnv MAIL_FROM_NAME ToolShare SetEnv STRIPE_SECRET_KEY sk_test_xxx SetEnv STRIPE_WEBHOOK_SECRET whsec_xxx SetEnv APP_URL http://toolsharevidhi.centralus.cloudapp.azure.com SetEnv TOOLSHARE_TMP_ROOT /var/www/toolshare/tmp ErrorLog ${APACHE_LOG_DIR}/toolshare-error.log CustomLog ${APACHE_LOG_DIR}/toolshare-access.log combined </VirtualHost>

Enable the site:

sudo a2enmod rewrite sudo a2dissite 000-default sudo a2ensite toolshare sudo systemctl reload apache2

Step 12: Run Migrations and Deployment Checks

In the same SSH session, export the same environment values once and run:

export DB_HOST=127.0.0.1 export DB_PORT=3306 export DB_NAME=tool_rental export DB_USER=toolshareapp export DB_PASS='YourStrongDbPasswordHere' export DB_CHARSET=utf8mb4 export MAIL_ENABLED=true export MAIL_HOST=smtp.gmail.com export MAIL_PORT=587 export MAIL_SECURE=tls export MAIL_USERNAME='yourgmail@gmail.com' export MAIL_PASSWORD='your_16_character_google_app_password' export MAIL_FROM_EMAIL='yourgmail@gmail.com' export MAIL_FROM_NAME='ToolShare' export STRIPE_SECRET_KEY='sk_test_xxx' export STRIPE_WEBHOOK_SECRET='whsec_xxx' export APP_URL='http://toolsharevidhi.centralus.cloudapp.azure.com' export TOOLSHARE_TMP_ROOT='/var/www/toolshare/tmp' cd /var/www/toolshare php scripts/run_migrations.php php scripts/predeploy_check.php

Step 13: Add HTTPS

Stripe and email links should use HTTPS. Install Certbot:

sudo apt -y install certbot python3-certbot-apache sudo certbot --apache -d toolsharevidhi.centralus.cloudapp.azure.com

After HTTPS is live, edit the Apache config and change:

SetEnv APP_URL http://toolsharevidhi.centralus.cloudapp.azure.com

to:

SetEnv APP_URL https://toolsharevidhi.centralus.cloudapp.azure.com

Then reload Apache:

sudo systemctl reload apache2

Step 14: Configure Stripe

  1. Use Stripe test mode.
  2. Set your webhook endpoint to:
    https://toolsharevidhi.centralus.cloudapp.azure.com/stripe_webhook.php
  3. Copy the webhook secret into `STRIPE_WEBHOOK_SECRET`.

Step 15: Final Smoke Test

  1. Open the homepage.
  2. Register a new user.
  3. Verify email from your phone.
  4. Run forgot password.
  5. Log in.
  6. Add a tool.
  7. Create a booking request.
  8. Approve the booking.
  9. Complete Stripe checkout in test mode.
  10. Open the receipt.
  11. Send one chat message.
  12. Create one support ticket.
  13. Walk one dispute flow.

Quick Troubleshooting

Issue Most likely cause Fix
Site opens but DB fails Wrong Apache `SetEnv` DB values Recheck `DB_HOST`, `DB_NAME`, `DB_USER`, `DB_PASS` and reload Apache
Email links go to localhost `APP_URL` still wrong Set it to the VM hostname or HTTPS hostname and reload Apache
Emails do not send Wrong Gmail app password or SMTP values Verify `MAIL_*` settings and use a Google App Password, not your normal password
Receipts fail Temp directory or permissions issue Check `/var/www/toolshare/tmp` and ownership for `www-data`
Uploads fail Folder permission issue Recheck `/var/www/toolshare/uploads` ownership and mode
